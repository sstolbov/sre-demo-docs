# Приложение в Kubernetes выдаёт ошибки, растёт время ответа: как проверить состояние и быстро восстановить

## 📋 Базовые проверки (первые 5 минут)
1. Проверка доступности сервиса
```bash
# Проверка из браузера
curl -I https://your-service.domain.com

# Проверка DNS
nslookup your-service.domain.com
# или
dig your-service.domain.com

# Проверка доступности снаружи кластера
ping your-service.domain.com
```
2. Вход в Grafana

Откройте браузер и перейдите по адресу: https://grafana.your-company.com

### 🚨 Признаки проблем (что видит пользователь)
| Признак |	Какой дашборд смотреть в Grafana |
|-|-|
|Сайт не открывается|	Kubernetes / Ingress Controller / Service Health|
|Страница грузится долго|	Response Times / Resource Usage|
|Ошибки 500|	Application Logs / Error Rates|
|Частично не работают функции|	Database / External Dependencies|
|Картинки не грузятся|	Static Content / CDN / PVC Usage|

## 📊 Пошаговая диагностика через Grafana
### Шаг 1: Общий статус кластера (1 минута)
Дашборд: Kubernetes / Cluster / Overview

Что смотреть:

CPU/Memory Usage - общая загрузка кластера (не должно быть >80%)

Node Status - все ноды в статусе Ready

Pod Health - количество подов с ошибками (должно быть 0)

Events - последние события в кластере (Warnings/Errors)

![Пример: Cluster Overview dashboard showing nodes and pods status]

Ключевые метрики для L1:

🔴 Красное - требует немедленного внимания

🟡 Желтое - нужно следить, возможно проблема

🟢 Зеленое - все хорошо

### Шаг 2: Проверка проблемного namespace (2 минуты)
Дашборд: Kubernetes / Namespace / <your-namespace>

Что смотреть:

Workload Status - все ли поды в статусе Running?

Pod Restarts - есть ли поды с частыми рестартами (>3 за час)

CPU/Memory Usage - не превышают ли limits?

Network I/O - нормальный ли трафик?

Где искать:

В верхнем селекторе выберите ваш namespace

Переключитесь на вкладку Pods или Workloads

### Шаг 3: Проверка логов (2 минуты)
Дашборд: Logs / Application / <app-name>

Что смотреть:

Error Rate - график ошибок (должен быть около 0)

Log Volume - нормальный ли объем логов?

Specific Errors - фильтруйте по "ERROR", "Exception", "Timeout"

Как фильтровать логи:

text
{namespace="your-namespace", pod=~".*your-app.*"} |~ "error|exception|timeout"
Шаг 4: Проверка производительности (1 минута)
Дашборд: Application / Performance / <app-name>

Что смотреть:

Request Duration (P95/P99) - время ответа (норма <500ms)

Request Rate - количество запросов (резкие падения = проблема)

Error Rate by Endpoint - какие URL дают ошибки?

Active Connections - не превышен ли лимит?

## 🛠 Быстрые решения для L1
### Решение 1: Если видите красные метрики по подам
Действие: Перезапустить проблемный deployment через интерфейс

Через kubernetes dashboard:

Откройте дашборд Kubernetes / Workloads

Найдите проблемный deployment

Нажмите на иконку меню (три точки)

Выберите Restart или Rollout restart

Подтвердите действие

ИЛИ через веб-интерфейс Kubernetes (Lens/OpenLens/Octant):

Откройте Lens/OpenLens

Перейдите в Workloads → Deployments

Найдите проблемный deployment

Нажмите правой кнопкой мыши → Restart

### Решение 2: Если растет время ответа
Действие: Проверить загрузку ресурсов и масштабировать при необходимости

Проверка в Grafana:

Откройте дашборд Resource Usage / Pods

Посмотрите CPU/Memory по каждому поду

Если приближается к limits → нужен рестарт или увеличение ресурсов

Принятие решения:

CPU >80% стабильно → нужен рестарт или увеличение CPU

Memory >90% и растет → возможна утечка памяти, нужен рестарт

Рестарты помогают временно → эскалировать на L2

### Решение 3: Если ошибки 500
Действие: Проверить зависимые сервисы

В Grafana:

Откройте дашборд Dependencies / Services

Проверьте статус БД, Redis, внешних API

Если зависимость недоступна → эскалировать на L2

## 📝 Шаблон для эскалации на L2
При передаче проблемы на L2 обязательно укажите:

```text
ВРЕМЯ: [когда началась проблема]
СЕРВИС: [название сервиса]
ЧТО ВИДИТ ПОЛЬЗОВАТЕЛЬ: [описание проблемы]

МЕТРИКИ ИЗ GRAFANA:
- Error rate: [график/значение]
- Response time (P95): [норма была X, стало Y]
- CPU/Memory: [значения]
- Рестарты подов: [было/стало]

ЧТО УЖЕ СДЕЛАНО:
- [ ] Проверены дашборды
- [ ] Перезапущены поды (время: ...)
- [ ] Проверены зависимости

ССЫЛКА НА GRAFANA DASHBOARD: [URL с примененными фильтрами]
ССЫЛКА НА LOGS: [URL с примененными фильтрами]
```
## 🎯 Чек-лист быстрого реагирования

## 🔍 Полезные дашборды в Grafana
|Название дашборда |	Что  показывает |	Когда смотреть |
|-|-|-|
|Kubernetes / Cluster |	Статус всего кластера, ноды |	При любой проблеме сначала сюда |
|Kubernetes / Namespace |	Поды, рестарты, ресурсы по namespace |	Когда знаете проблемный сервис |
|Application / Performance |	Время ответа, RPS, ошибки |	При жалобах на медленную работу |
|Application / Logs |	Логи приложения	| При ошибках 500 |
|Database / Overview |	Статус БД, медленные запросы	| Когда проблемы с сохранением данных |
|Ingress / Overview |	Трафик, ошибки на входе	| Когда сайт совсем не открывается |
